# Методы программирования приложений

В данном разделе рассматриваются основные подходы к созданию приложений, классифицированные по типу интерфейса и среды исполнения.

## 1. Консольные приложения
Программы, использующие текстовый интерфейс (CLI) для взаимодействия с пользователем.

- **Интерфейс:** Текстовый ввод/вывод (stdin/stdout).
- **Особенности:**
  - Минимальное потребление ресурсов.
  - Простота автоматизации через скрипты.
  - Отсутствие графической оболочки.
- **Применение:** Системные утилиты, серверные службы (daemons), средства администрирования, учебные программы.

## 2. Оконные Windows-приложения
Программы с графическим интерфейсом пользователя (GUI), ориентированные на работу в среде ОС Windows.

- **Интерфейс:** Окна, формы, кнопки, меню.
- **Особенности:**
  - Событийно-ориентированная модель (Event-driven).
  - Использование API операционной системы (WinAPI) или высокоуровневых оберток.
- **Технологии (Microsoft):**
  - **Windows Forms:** Классическая библиотека классов для разработки GUI.
  - **WPF (Windows Presentation Foundation):** Современная платформа на базе XAML и DirectX.
  - **WinUI / UWP:** Платформы для универсальных приложений.

## 3. Web-приложения
Клиент-серверные приложения, где клиентом выступает браузер, а взаимодействие происходит по сети.

- **Интерфейс:** HTML-страницы, отображаемые в браузере.
- **Особенности:**
  - **Кроссплатформенность:** Работают на любой ОС с браузером.
  - **Доступность:** Не требуют установки (достаточно URL).
  - **Архитектура:** Четкое разделение на Frontend (клиентская часть) и Backend (серверная логика).
- **Механизм:** Работают по протоколу HTTP/HTTPS (Запрос -> Обработка на сервере -> Ответ).
