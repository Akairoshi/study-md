# Библиотеки в программировании

**Библиотека** — это сборник подпрограмм, классов и объектов, предназначенный для использования в разработке программного обеспечения, но не являющийся самостоятельной исполняемой программой.

## Основные цели
- **Повторное использование кода (DRY):** Использование готовых решений для типовых задач (математика, ввод/вывод, работа с сетью).
- **Модульность:** Разделение программы на логические блоки.
- **Оптимизация:** Уменьшение размера исходного кода основного приложения.

## Типы библиотек

### 1. Статические библиотеки
- **Расширение:** `.lib` (Windows), `.a` (Linux/Unix).
- **Принцип работы:** Код библиотеки копируется внутрь исполняемого файла на этапе компиляции (линковки).
- **Плюсы:** Исполняемый файл автономен и не требует внешних файлов при запуске.
- **Минусы:** Увеличивается размер программы; при обновлении библиотеки нужно перекомпилировать всю программу.

### 2. Динамические библиотеки
- **Расширение:** `.dll` (Windows), `.so` (Linux/Unix).
- **Принцип работы:** Код подгружается в память только во время выполнения программы. Одна библиотека может использоваться несколькими программами одновременно.
- **Плюсы:** Экономия оперативной памяти; возможность обновления логики без пересборки основного приложения.
- **Минусы:** Проблема зависимостей («DLL Hell»), необходимость наличия файла библиотеки на целевой машине.
