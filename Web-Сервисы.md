# Web-сервисы

**Web-сервис** — это программная система, предназначенная для взаимодействия между компьютерами через сеть (обычно интернет). Она позволяет приложениям обмениваться данными независимо от языка программирования и платформы.

## Ключевые характеристики
- Идентифицируется через URI (Uniform Resource Identifier).
- Описывает свои интерфейсы на языке XML или посредством спецификаций JSON.
- Использует стандартные интернет-протоколы (HTTP, SMTP и др.).

## Основные архитектурные стили

### 1. SOAP (Simple Object Access Protocol)
Протокол обмена структурированными сообщениями.
- **Формат данных:** Строго XML.
- **Особенности:** Высокая стандартизация, встроенная обработка ошибок, поддержка транзакций и высокий уровень безопасности (WS-Security).
- **Применение:** Банковский сектор, корпоративные системы (Enterprise), где критична надежность контрактов.

### 2. REST (Representational State Transfer)
Архитектурный стиль взаимодействия компонентов распределенного приложения.
- **Формат данных:** Чаще всего JSON (реже XML).
- **Принцип работы:** Основан на методах протокола HTTP:
  - `GET` — получение данных.
  - `POST` — создание ресурса.
  - `PUT/PATCH` — обновление данных.
  - `DELETE` — удаление.
- **Особенности:** Легковесность, простота, кэшируемость, высокая производительность.
- **Применение:** Современные веб-сайты, мобильные приложения, микросервисы.
